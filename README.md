# DevBrain

DevBrain — это мощный и интуитивно понятный менеджер ссылок с поддержкой тегов, поиска и аналитики. Он позволяет организовать ваши закладки, статьи, видео и другие ресурсы в одном месте.

## Особенности

- **Теги:** Назначайте теги ссылкам для быстрой группировки и фильтрации.
- **Поиск:** Быстрый поиск по заголовкам, URL и тегам.
- **Управление:** Добавляйте, редактируйте и удаляйте ссылки.
- **Статистика:** Просматривайте количество сохранённых ссылок и тегов.
- **Безопасность:**
  - Аутентификация с использованием JWT.
  - Защита от атак (Rate Limiting, CORS, безопасные заголовки).
  - Хеширование паролей (Argon2id).
- **Технологии:** Go (backend), HTML/CSS/JavaScript (frontend), SQLite (или PostgreSQL).
- **Деплой:** Простой деплой на Render.com (пример конфигурации включён).

## Установка и запуск (локально)

1.  **Убедитесь, что у вас установлен Go** (версия >= 1.21).
2.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/skywaJlker192/DevBrain.git
    cd DevBrain
    ```
3.  **Создайте файл `.env`** в корне проекта, основываясь на `.env.example`. Укажите настройки базы данных (SQLite или PostgreSQL) и секретные ключи.
4.  **Установите зависимости (опционально, Go обычно делает это автоматически):**
    ```bash
    go mod tidy
    ```
5.  **Запустите сервер:**
    ```bash
    go run ./cmd/server
    ```
    Сервер запустится на `http://localhost:8080`.

## Деплой на Render.com

1.  Создайте аккаунт на [Render.com](https://render.com).
2.  Создайте новый **Web Service**.
3.  Подключите репозиторий GitHub.
4.  Укажите:
    -   **Build Command:** `go build -o server ./cmd/server`
    -   **Start Command:** `./server`
5.  В разделе **Environment Variables** укажите переменные из вашего `.env` файла (DB_HOST, JWT_SECRET и т.д.).
6.  Нажмите **Create Web Service**.

## Структура проекта

-   `cmd/server/main.go`: Точка входа в приложение.
-   `internal/config/`: Конфигурация приложения.
-   `internal/handlers/`: Обработчики HTTP-запросов.
-   `internal/repository/`: Работа с базой данных.
-   `internal/security/`: Логика безопасности (JWT, хеширование).
-   `static/`: Статические файлы (HTML, CSS, JS).
-   `migrations/`: SQL-скрипты для миграций базы данных.
-   `go.mod`, `go.sum`: Зависимости Go-модуля.
